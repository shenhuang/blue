## 1. 战斗系统设计目标

1. **轻量、文字友好**：不引入网格、不引入距离段位、不引入卡组构筑。
2. **资源紧张**：每一回合的氧气都在走，每一次出手都是真实代价。
3. **多解 / 多胜利路径**：不止"打死它"才算赢，潜行、驯服、谈判、惊吓驱赶都算成功。
4. **克苏鲁敌人是另一回事**：物理对它们部分无效；« 2026-07-10 理智系统移除：理智轴已删，克苏鲁敌人回归物理单轨；"物理部分无效 + 非物理胜利路径"的差异化改由弱点/免疫/victoryConditions 承载·见 §2.3 TODO »。
5. **能承载从鲨鱼到巨物的全部尺度**：一套机制兼容写实捕食者与不可名状之物。

## 2. 核心模型

### 2.1 行动经济：双资源直读

抛弃 AP/法力等抽象数字。每个行动**直接读取消耗 `(体力, 氧气回合数)`**：

```
行动 := { name, costStamina, costOxygenTurns, effect, requires? }
```

`costOxygenTurns` 含义：执行该行动时，玩家氧气按"N 个标准下潜回合"扣除。这把战斗与下潜统一在了同一氧气消耗模型下——战斗是耗氧的，剧烈战斗加速窒息。

战斗中的氮气浓度也照常按当前深度累积，但比下潜节点稍快（剧烈呼吸加速氮气吸收），系数建议 ×1.5。

### 2.2 体力与失能

- 体力 0 ≠ 死亡，而是 **失能（Incapacitated）**
- 失能后下回合开始无法主动行动，敌人攻击伤害 +50%，连续 2 回合无救助 → 死亡
- 失能可被"急救包""战友救助"事件打断（后期内容）

体力低于 30% 时，所有需要 `costStamina ≥ 8` 的行动**附加 1 回合氧气消耗**（喘息）。

### 2.3 伤害轨道（物理单轨）

> « 2026-07-10 理智系统移除 »：**理智伤整轨作废**——`run.stats.sanity` / `sanityHp` / `EnemyAttack.sanityDamage` / `immunity 'sanity'` 全删，战斗现为**物理单轨**。下表理智伤行、及"两轨都打"设定仅存为历史设想；凝视/低语/几何错乱这类"非物理威胁"如需保留，改由物理伤 + 负伤 / 环境压力表达（注：负伤"惊惧"依赖已删的 `sanity` 伤种，也一并作废·见负伤 SPEC §6）。« TODO(作者)：克苏鲁敌人失去理智轨后的差异化手段（弱点门 / 非物理胜利路径 / 纯氛围）待定 »

| 伤害类型 | 来源 | 防御方式 | 归零结果 |
|---|---|---|---|
| 物理伤 | 牙、爪、撞击、刺 | 潜水服装甲、闪避 | 体力 0 → 失能 |


### 2.4 没有位置 / 距离维度

战斗只发生在"你与它们"之间。位置维度的策略性由**武器属性**与**敌人特性**替代：

- 武器有"性格"（噪声、攻击范围标签、最佳条件）
- 敌人有"姿态"与"行为模式"（见 §4）

## 4. 敌人系统

### 4.1 敌人数据 Schema

```typescript
type Enemy = {
  id: string;
  name: string;
  tier: 'realistic' | 'uncanny' | 'cosmic';
  
  // —— 基础属性 ——
  hp: number;               // 体力 / 物理生命值
  defense: number;          // 防御力（原 armor 改名 2026-07-10·战斗系统改版 #290）：resolveDamage 物理减伤基值
  speed: number;            // 与玩家"敏捷"对比，决定先手
  // （evasion/hitBonus 已随命中率系统整套删·战斗改「必中」·#290/#291·别再往这个 schema 加命中相关字段）
  
  // —— 威胁与行为 ——
  threat: number;           // 初始威胁度，影响多敌时的攻击概率
  hostility: 'aggressive' | 'territorial' | 'passive' | 'predatory' | 'curious' | 'reverent';
  stance: 'unaware' | 'alerted' | 'attacking' | 'enraged' | 'fleeing';
  
  // —— 攻击模式 ——
  attacks: EnemyAttack[];
  aiPattern: string;        // 引用 AI 模板：'aggressor' | 'flanker' | 'caster' | 'observer' | 'crowd' ...
  
  // —— 伤害与抗性 ——
  physicalDamage: [min, max];
  weakness?: ('light' | 'sound' | 'salt' | 'cold' | 'iron' | 'fresh-water' | 'name')[];
  immunity?: ('physical' | 'electricity' | 'fire')[];
  
  // —— 战利品与结算 ——
  loot: LootTable;
  victoryConditions: ('kill' | 'flee' | 'tame' | 'scare' | 'commune')[];
  loreEntry?: string;       // 击败/驯服后解锁的图鉴条目
};
```

### 4.2 敌人姿态状态机

```
unaware ─发出声响/视线─▶ alerted
   │                        │
   │                        ▼
   │                    attacking ◀── 玩家攻击
   │                        │
   │           攻击次数/受伤▼
   │                    enraged
   │                        │
   │              HP↓ ▼
   └─────潜行成功─────▶  fleeing
```

不同 hostility 决定初始姿态和转移条件：
- `aggressive`：默认 attacking
- `territorial`：unaware → 接近后 alerted → 越界 attacking
- `predatory`：从 unaware 突袭 attacking（潜伏）
- `passive` / `curious`：始终在 alerted，受攻击才进 attacking
- `reverent`：克苏鲁仪式型，从 unaware 缓慢进入 attacking，玩家可"交谈"

### 4.3 敌人 AI 行为模板（5 个起步）

| AI 模板 | 行为 | 适用敌人 |
|---|---|---|
| aggressor | 每回合用最高伤害技能锁定玩家 | 鲨鱼、海蛇 |
| flanker | 隔回合行动，行动时附加附条件加成（出血/缠绕） | 章鱼、巨型墨鱼 |
| caster | 玩家攻击时 50% 概率"逃入水中"位移取消攻击（« 2026-07-10 理智系统移除 »：远程伤改物理或纯位移干扰） | 深潜者祭司 |
| observer | 不直接攻击，每回合 +1 自身或同场敌人威胁度，被击破即视为胜利 | 巨大的眼睛、远处的轮廓 |
| crowd | 多个低 HP 实体，行动量少但数量碾压，AoE 高效 | 蟹群、寄生体 |

### 4.4 三档敌人原型示例

#### 写实层（0–60m）

- **暗礁鲨**：aggressive / aggressor，HP 高、armor 中、纯物理伤、噪声引来
- **巨型石斑**：territorial，HP 中、领地反击、被驱赶后离去
- **海鳗**：predatory，从洞穴突袭、命中下降的玩家必失误
- **狮子鱼**：passive，碰它才伤、伤口持续扣体力（毒）

> « 2026-07-10 理智系统移除 »：下列**异样层 / 克苏鲁层原型里一切"扣理智 / 理智双伤 / sanityHp 归零"效果均已作废**（历史设想保留）。落库时非物理威胁改由物理伤 + 负伤 / 环境压力表达；"被理解→撤退"的克苏鲁胜利路径失去 sanityHp 载体（见 §2.3 TODO）。

#### 异样层（50–120m）

- **错位的鱼群**：crowd，看起来正常但游动方向"全错了"（原扣理智效果已删·改物理小扰或纯氛围）
- **回声**：observer，听到的是自己几秒前发出的声音、但内容不一样，被"喊破"即消失
- **变异巨章**：flanker，缠绕扣体力

#### 克苏鲁层（100m+）

- **深潜者初信徒**：aggressor，物理，怕"咏唱"（玩家护身符 lv3+ 触发）
- **沉默的合唱**：observer + caster，多个轮廓远处出现（原持续扣理智已删·改物理/纯氛围）
- **它的一部分**：boss tier，巨大触手只是它的一节，本体不可见。无法被杀，只能"被见到的一部分被切断"——（原 sanityHp 归零结束遭遇已删·结束条件需换门）

## 5. 战斗前：遭遇事件

战斗不是从"战斗界面"突然开始，而是从一个**遭遇事件**开始。每场遭遇先弹一个事件页：

```
[一只大暗礁鲨从礁石阴影中游出，盯着你]

> 慢慢后退，不发出声响（潜行检定 vs 鲨鱼威胁）
> 端起鱼叉，先发制人（突袭，进入战斗时附加暴击）
> 把刚捞起的鱼肉扔出去（消耗：鲜鱼 ×1，敌人转向追逐鱼肉）
> 拍打装备故意发出声音吓走它（改体力/装备门〔原理智门已删·« 2026-07-10 »〕，鲨鱼威胁 ≤ X 时成功）
> 直接战斗
```

设计要点：

- 每个敌人 schema 中可以定义 `encounterOptions`，复用事件 schema 的 EventOption
- **潜行**消耗氧气而非体力（屏息）；失败概率与发出过的声响/装备灯光相关
- **交涉/驯服/惊吓**作为非战斗胜利路径，需要特定装备或属性阈值
- 失败的非战斗选项 → 直接进入战斗，敌人姿态升至 `attacking` 或 `enraged`

## 6. 胜利路径与战利品

### 6.1 多种胜利条件

每个敌人的 `victoryConditions` 决定哪些路径可走，配套奖励：

| 路径 | 触发 | 奖励 |
|---|---|---|
| 击杀 (kill) | HP 归 0 | 满战利品表、肉/部件类材料 |
| 击退 (flee) | 敌人 stance = fleeing 后玩家脱战 | 残破战利品（一半概率）、解除遭遇 |
| 潜行/脱身 (escape) | 屏息潜逃成功或敌人 unaware 中脱离 | 无战利品但**保全氧气**、不进入图鉴 |
| 驯服 (tame) | 多步事件、特定道具/装备 | 解锁可携带的"伙伴生物"小机制（后期） |
| 惊吓 (scare) | 敌人 morale 归零 / 触发特定弱点 | 区域中此敌人种群冷却若干次下潜不再出现 |
| 交流/契约 (commune) | 克苏鲁敌人专属，玩家持有特定知识（« 2026-07-10 理智系统移除 »：sanityHp 触发已删，commune 只剩"持有特定知识"一路） | 知识碎片（用于解锁港口教堂线/隐藏剧情） |

### 6.2 战利品表 (LootTable)

```typescript
type LootTable = {
  guaranteed?: LootRoll[];
  rolls: { weight: number; item: string; qty: [min, max] }[];
  rollCount: number;             // 几次抽取
  victoryModifier?: {            // 不同胜利路径加成
    [path: string]: number;      // 例：{ scare: 0.5, tame: 0 }
  };
};
```

战利品里区分：
- **直接金币**：杂物商人愿意收
- **材料**：用于港口装备升级（鲨鱼牙、墨鱼囊、扭曲的角）
- **知识碎片**：用于解锁港口高阶升级 + 图鉴
- **特殊物品**：链式事件触发器（"会发光的眼珠"打开下次下潜的特定事件）

## 7. 战斗中的多敌处理

### 7.1 每场战斗有一个 EnemyParty

```typescript
type EnemyParty = {
  members: Enemy[];                    // 1–4 个
  joinRules?: {                        // 战斗中其他敌人加入的规则
    triggerOnNoise?: { threshold: number; addFromPool: string[]; max: number };
  };
};
```

> "噪声触发增援"是水下战斗最有压力的设计——玩家鱼叉打中鲨鱼会引来更多鲨鱼，电击放出会引来深处某个东西。

### 7.2 仇恨与目标选择

每个敌人独立维护自己的 `aggro`（仇恨）：
- 玩家攻击该敌人 → 它的 aggro 上升
- 玩家用诱饵/挑衅 → 指定 aggro 转移
- 玩家"低姿态"（连续若干回合不攻击）→ 所有 aggro 缓慢下降，部分敌人进入 fleeing

每个敌人独立决定行动：
- 高 aggro 的敌人优先攻击玩家
- 低 aggro 敌人可能：等待、互相干扰（章鱼缠住鲨鱼一回合）、撤退、围观（observer 类）

### 7.3 玩家行动的"群体语义"

某些行动天然 AoE：
- 电击器：本回合该深度全部敌人受眩晕检定
- 信号弹/烟雾弹：全场效果
- 调息：仅对玩家自身

非 AoE 行动需要选目标，UI 设计应让选目标占用最少操作（按数字键或紧凑列表）。

## 9. 战斗结束后

### 9.1 战斗结束 = 进入"战后节点"

- 如果就地继续探索：仍在原节点，可以选择"搜刮""调查""休息（消耗回合补体力）""离开"
- 如果是潜行/脱身：玩家不留在原地，直接进入下一节点选择
- 战利品在战后节点显示，玩家选择哪些放进背包（受负重限制）

### 9.2 图鉴解锁

每个敌人首次以特定胜利路径处理后，解锁对应 `loreEntry`。同一敌人不同路径解锁**不同条目**——驯服深海生物的图鉴可能比击杀的图鉴透露更多真相。

图鉴本身是港口"打捞行会"升级线的次级奖励：解锁条目越多，下次出海可以"预知"节点类型。

## 12. 开放问题

1. **失能后被处决的演出**：是直接死亡还是给玩家"最后挣扎"的紧急按键（消耗高额道具/体力反杀）？
2. **首回合手感**：玩家是否总是先手？还是 speed 比较决定先手？
3. **氮气在战斗中是否有"屏息减速"机制**：闭气可减缓氮气累积但加速体力消耗？
4. **图鉴条目的克苏鲁化**：高深度敌人的图鉴是否本身就是"被污染的文字"？（« 2026-07-10 理智系统移除 »：理智门已删·若保留此设想需换门）
5. **战斗音效**：纯静态站要不要尝试 Web Audio 做心跳/呼吸/水声？影响沉浸感巨大。
6. **战利品的"易腐"**：生物部件是否会在背包停留若干次下潜后失效？促使玩家及时返港。
