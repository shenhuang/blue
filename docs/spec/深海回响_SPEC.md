## 1. 设计支柱

1. **每一次下潜都是一次取舍**：氧气是时间，深度是回报，氮气是债务。
2. **死亡是叙事而非惩罚**：尸体留在海底，下一次出航你可能成为别人的传说。
3. **越深越不可知**：写实 → 异样 → 克苏鲁的渐进过渡，现仅由深度驱动（« 2026-07-10 理智系统移除 »：原双驱动·理智轴已删·另设机制 TODO 见顶部 banner）。
4. **数据驱动可扩展**：所有事件、装备、升级以 JSON 配表，方便后续大规模内容填充。

## 2. 核心循环

```
出港 → 选择下潜海域 → 随机生成节点图（浅海池 → 深海池过渡）
     ↓
进入节点（回合制：每回合消耗氧气，深度叠加氮气，事件可额外加成）
     ↓
触发事件 → 选项检定 → 属性增减 / 物品获得 / 链式剧情
     ↓
判断：继续深入？ 选择上浮？ 应急上浮？
     ↓
上浮（含减压停留 mini-loop） → 回港
     ↓
结算：变卖宝藏（金币）、记录探索深度（建设值）
     ↓
升级港口、购买装备 → 再次出航
```

死亡时，本次身上的宝藏与一次性物资留在海底；探索程度转化为**港口建设值**，永久投资升级树。

## 3. 四属性系统

| 属性 | 起始 | 上限 | 主要消耗 | 主要恢复 | 备注 |
|---|---|---|---|---|---|
| 体力 Stamina | 100 | 装备决定 | 挣扎、战斗、强游 | 休息回合、补给 | 影响"逃跑/拼搏"类选项检定 |
| 氧气 Oxygen | 由气瓶决定 | 由气瓶决定 | 每回合按深度系数消耗 | 仅靠备用气瓶/事件 | 归零 = 死亡 |
| 氮气浓度 Nitrogen | 0 | 100 | 深度持续累积 | 上浮+减压停留、混合气 | 过高 → 减压病；氮气现只喂减压债 |

### 3.1 属性耦合

- **体力 ↓** → 动作消耗氧气加倍

### 3.2 UI 关键预警

属性面板必须显示两个派生值，让玩家明确感知风险边界：

- **氧气剩余回合数**（按当前深度基础消耗预估）
- **安全上浮所需回合数**（按当前氮气浓度的减压方案预估）

当后者 > 前者，UI 标红警告 —— 玩家知道自己已经"过界"。

## 4. 上浮与减压系统

### 4.1 三种上浮方式

| 类型 | 触发 | 代价 |
|---|---|---|
| 正常上浮 | 玩家主动选择 | 经过若干上浮节点，可能触发事件 |
| 应急上浮 | 任意时刻可触发 | 跳过减压停留，**几乎必得减压病** |
| 强制上浮 | 氧气≤危险线、特定事件 | 等同应急上浮，无玩家选择空间 |

### 4.2 减压病分级

| 分级 | 触发条件 | 后果 |
|---|---|---|
| I 型 皮肤型 | 轻微氮气超标 | 回港治疗费扣金币，无长期后果 |
| II 型 关节型 | 中度氮气超标 | 下次下潜起始体力 −30% |
| III 型 神经型 | 重度氮气超标 | « 2026-07-10 理智系统移除 »：原后果依赖已删的理智轴、失去载体。« TODO(作者)：III 型改扣何种资源（体力/氧上限）或下架 » |
| IV 型 致命 | 极重 + 高深度 | **当场死亡**，尸体保留全部战利品 |

> MVP 阶段只实现 II 型与 IV 型，I/III 型后续扩展。

## 6. 探索与节点系统

### 6.1 节点图生成

每次出海生成一张**有向图**，节点带深度属性。事件池按深度分段（`shallow` / `coastal` / `twilight` / `midnight` / `abyssal`），随推进自然过渡。

每个节点暴露 2–3 个下一节点选项，玩家选择即推进。某些节点有特殊标签：

- **上浮口**：可发起正常上浮
- **休息点**：可消耗回合恢复体力/部分氧气
- **沉没遗迹**：固定特殊事件链
- **尸体定位**：与玩家历史死亡点匹配，触发尸体回收

### 6.2 回合制时间粒度

每回合自动结算：

```
氧气 -= baseOxygenCost × depthFactor × staminaFactor × weightFactor
氮气 += baseNitrogenGain × depthFactor
```

某些事件附加 `oxygenTurnCost: 3`，代表"这次行动额外消耗 3 回合氧气"。

## 7. 死亡、尸体与元进度

### 7.1 死亡时

- 本次身上**所有战利品与一次性物资**留在海底
- 本次永久装备：**留在海底**（需找回尸体）
- 金币：**全部留在海底**（除非已存入港口银行，见 §9.2）
- 探索程度 → 转化为**港口建设值**：
  ```
  建设值 = 最深深度系数 + 触发事件数 × α + 完成事件链 × β
  ```

### 7.2 尸体系统

- 尸体在**死亡区域相近深度**生成，下次出海有概率匹配进节点池
- 节点选择时给出文字提示："这一带似乎有熟悉的东西…"
- 回收时受限于当前背包容量与负重
- 尸体随"水下时间"（实际下潜次数）逐步腐烂：物品失效、信息丢失
- 隐藏机制：尸体在同一海域累积过多 → 触发"亡者之径"事件链（吸引深海某物）

### 7.3 港口建设值升级树

四条独立线，玩家需在有限建设值下取舍：

| 线 | 主题 | 典型升级 |
|---|---|---|
| 船坞 | 出海起点 | 起始水深、跳过浅水跑图、出海前补给 |
| 气瓶库 | 呼吸方案 | 氧气上限、混合气解锁、备用气瓶 |
| 打捞行会 | 信息与回收 | 尸体定位、宝藏地图、节点预览 |
| 教堂/灯塔 | 心理建设 | « 2026-07-10 理智系统移除 »：原三项升级皆挂已删的理智轴、失去载体（灯塔机制另见声呐/深水区 SPEC） |

## 8. 装备与背包

### 8.1 装备槽（5 槽固定）

| 槽位 | 关键效果 |
|---|---|
| 气瓶 | 氧气上限、混合气类型 |
| 潜水服 | 体力上限、保暖（深度负面修正抗性） |
| 灯具 | 节点视野（提前看清下一节点类型）、影响特定事件 |
| 副手工具 | 鱼叉/切割器/相机等，影响事件选项 |
| 护身符 | « 2026-07-10 理智系统移除 »：装备 `sanityResist` 已删·护身符槽失去 niche。« TODO(作者)：护身符改赋何效或下架该槽 » |

装备每个槽位有等级（1–5）+ 可选词缀，词缀通过事件、稀有掉落、港口高级升级获得。

> MVP 阶段先做前 4 槽，护身符在克苏鲁层扩展时引入。

### 8.2 背包

- **格子上限**：基础 8 格，港口升级可至 16 格
- 高价值物品占多格（一只完整的鱼叉可能占 2 格）
- **负重**：所有物品累积负重影响上浮速度与每回合氧气消耗
- 取舍设计：拿一件 4 格的稀有宝物 vs 多塞 2 件中级宝物

## 9. 经济系统

### 9.1 双货币

- **金币**：通过变卖战利品获得，购买装备、消耗品
- **建设值**：通过探索程度获得，升级港口

两套货币独立运作，避免"刷钱通关"的退化路径。

### 9.2 港口银行（升级解锁）

港口升级解锁银行后，玩家可在出海前**存入部分金币**，存入部分死亡后保留。
鼓励"分批冒险"策略，但要交手续费。

### 9.3 消耗品

- 额外氧气瓶（途中补气）
- 急救包（恢复体力）
- 镇静剂已作废（原恢复理智·随理智系统删除·« 2026-07-10 »）·改回何种资源见战斗 SPEC §3.3 TODO
- 修复包（损坏装备修复）

消耗品占背包格，深海下决定带几个是核心策略。

## 10. 战斗系统（占位）

MVP 阶段：生物事件走选项检定，无独立战斗系统。

后续设计：轻量回合制独立战斗，不沿用 catacomb 战斗机制。

- 战斗中氧气仍持续消耗，强迫"快速解决"心理压力

## 12. 事件数据 Schema

```typescript
type DiveEvent = {
  id: string;
  // —— 触发条件 ——
  depthRange: [min: number, max: number];
  zoneTags?: ('wreck' | 'cave' | 'reef' | 'abyss' | 'ruins')[];
  weight: number;                                 // 抽取权重
  cooldown?: number;
  oncePerRun?: boolean;
  oncePerSave?: boolean;                         // 剧情向事件
  prereqEventIds?: string[];
  prereqFlags?: string[];

  // —— 文本与呈现 ——
  title: string;
  body: string;                                  // 模板变量 {player.name} {depth}
  tone: 'realistic' | 'uncanny' | 'cosmic';    // 影响 UI 配色、字体故障

  options: EventOption[];
};

type EventOption = {
  label: string;
  visibleIf?: Condition;                         // 需要装备/技能/物品
  check?: {
    stat: 'stamina' | 'nitrogen' | 'oxygen';
    dc: number;
    onSuccess: Outcome;
    onFailure: Outcome;
  };
  outcome?: Outcome;
};

type Outcome = {
  text: string;
  deltas?: Partial<Record<Stat, number>>;
  oxygenTurnCost?: number;                       // 额外消耗 N 回合氧气
  loot?: LootRoll[];
  applyFlags?: string[];
  triggerEventId?: string;                       // 链式事件
  endDive?: 'forceAscend' | 'death';
};
```

### 12.1 数据流

```
designer 编辑 events.json / xlsx
   ↓ （可选 dataconv 脚本）
   游戏读取 JSON
   ↓
   按当前 depth/flags 过滤、按 weight 抽取
   ↓
   渲染事件 → 玩家选项 → 结算 → 链式触发
```

## 13. 技术栈

- **前端**：TypeScript + Vite + 简洁 DOM/Canvas 渲染
- **数据**：JSON 直接手写（事件多分支适合 JSON）；如有协作需求加 xlsx → JSON 转换层
- **状态管理**：单一 GameState store，所有事件都是 `(state, choice) → newState` 的纯函数
- **持久化**：LocalStorage 存档（多存档槽）+ JSON 导出（便于备份和分享）
- **部署**：纯静态站点，可发到 GitHub Pages / Gitee Pages，无后端

## 附录 A · 默认数值预案（待平衡）

```
起始 体力 100 / 100
起始 氧气 60 回合（一级气瓶）
起始 氮气 0 / 100

每回合基础氧气消耗 1
深度系数 = 1 + depth / 50  （30m 时为 1.6，100m 时为 3.0）
深度氮气增量 = depth / 30   （30m 时每回合 +1，100m 时每回合 +3.3）

氮气阶段：
  0–40:    安全
  40–60:   需 1 次减压停留
  60–80:   需 2 次减压停留
  80–95:   需 3 次减压停留
  95+:     强制减压
```

所有数值在 M4 内测前都视为占位，按测试反馈大幅调整。
