## 1. `zone.gates` 待填 schema（作者照此填）

**命名档 → 数字映射（占位·统一调时一次性定）**：`无=省略该 sense 或该档` · `稀≈?` · `中≈?` · `密≈?`（数字 defer）。`hiddenRatio` 同理只给曲线形状（见 C.3）。

```jsonc
// 加进 zones.json 对应 zone 对象。省略某档＝该档不撒；省略某 sense＝该 sense 全不撒。
"gates": {
  "lamp":  { "deep": <密级>, "mid": <密级>, "shallow": <密级>, "hiddenRatio": <0..1> },
  "sonar": { "deep": <密级>, "mid": <密级>, "shallow": <密级>, "hiddenRatio": <0..1> }
}
```

- **只填 zone 的 depthRange 够得到的档**（够不到的档写了也永远撒不出·见各 zone「可达档」）。
- `reason` **不在** `zone.gates` 里——它挂在**具体节点 / 整潜门**（`NodeGate.reason`），撒点侧只出中性兜底门；语义 `reason` 走内容侧（§4 override / 事件数据）。故本文的 `reason` 行 = 给作者做「这类门想让玩家读到什么」的占位参考，实装时落到内容。

## 2. 参与 zone 的逐个配置（推荐参与 4 座）

> 密度全用命名档。「可达档」= 该 zone depthRange 机械上能撒门的档（其余档即便填也 0）。

### 2.1 `zone.blue_caves` — 蓝洞群（**旗舰 demo · 灯 + 声呐 双感**）
- depthRange `[12,55]` → **可达档：mid(50-55) · shal(25-49)**（<25 段不撒）。maze · canFreeAscend:false。
- 定位：给作者做「灯门 2 格 + 声呐门 2 格」的**四格样板**（SPEC §2 那张 2×2）。双感、密度中等、浅端全 locked（诚实预告）、往深一点点起 hidden。

| sense | deep(≥90) | mid(50-89) | shallow(25-49) | hiddenRatio |
|---|---|---|---|---|
| lamp  | —（够不到） | 中 | 稀 | 浅≈0 → 中档略升 |
| sonar | —（够不到） | 稀 | 稀 | 浅≈0 → 中档略升 |

- `lamp.reason`（占位）：`太暗，看不清——需要灯`（兜底即可·蓝洞天然黑）。
- `sonar.reason`（占位）：`岩体拐过去没了视线，得扫一记声呐才认得清`（成因＝洞壁遮挡·非浑浊）。

### 2.2 `zone.open_midwater` — 远洋中层（**主探索区 · 声呐为主 + 灯为辅**）
- depthRange `[55,85]` → **可达档：mid 全段**（deep 够不到·shal 够不到）。layered · canFreeAscend:true（开阔水）。
- 定位：主力探索区、声呐的主场（中层无参照物·声呐＝找到「会折光的薄膜」那类岔口）。灯只做少量点缀。

| sense | deep(≥90) | mid(50-89) | shallow(25-49) | hiddenRatio |
|---|---|---|---|---|
| sonar | —（够不到） | 中 | —（够不到） | 中档中等（这里全是 mid·给一档 hidden 岔口） |
| lamp  | —（够不到） | 稀 | —（够不到） | 中档略有 |

- `sonar.reason`（占位）：`空水里没有边界，声呐扫一记才分得出哪儿有东西`。
- `lamp.reason`（占位）：`光晕外面还有一层，需要灯照到`。
- **注意 C.2**：本区是否叠**整潜声呐门**（`diveModifier.gate:{sonar,locked}`）另议——见 C.2（推荐**不**叠整潜、只 per-node）。

### 2.3 `zone.vent_trench` — 海沟热液场（**深区 · 灯为主 + 声呐为辅**）
- depthRange `[85,118]` → **可达档：deep(90-118) · mid(85-89 一小段)**（shal 够不到）。maze · canFreeAscend:false。
- 定位：深、黑、热液柱林立——灯是硬需求；声呐辅助（黑烟 / 悬浮矿物搅乱视线的岔口）。深档密度最高（越深越藏·C.3）。

| sense | deep(≥90) | mid(50-89) | shallow(25-49) | hiddenRatio |
|---|---|---|---|---|
| lamp  | 密 | 中 | —（够不到） | 深档最高（深⇒更藏） |
| sonar | 中 | 稀 | —（够不到） | 深档偏高 |

- `lamp.reason`（占位）：`黑烟把光吃光了，得靠自己的灯`。
- `sonar.reason`（占位）：`矿物悬在水里晃眼，扫一记声呐才看得清结构`。

### 2.4 `zone.whalefall` — 鲸落（**深区 · 灯为主**）
- depthRange `[80,110]` → **可达档：deep(90-110) · mid(80-89 一小段)**。layered · canFreeAscend:true。
- 定位：巨兽尸骸、拆解生态——深且暗，灯为主；声呐可选留很稀一撮或不开（作者定）。

| sense | deep(≥90) | mid(50-89) | shallow(25-49) | hiddenRatio |
|---|---|---|---|---|
| lamp  | 密 | 中 | —（够不到） | 深档最高 |
| sonar | 稀（或 无） | 无 | —（够不到） | 深档偏高 |

- `lamp.reason`（占位）：`骨架的阴影里什么都看不清，需要灯`。
- `sonar.reason`（占位·若开）：`尸肉挡住了后面，扫一记才知道通不通`。

## 3. 26 洞穴 zone — **按 tag 模板**（不逐 zone 手配 · 免 churn · C.1）

## 4. 内容侧 `reason` 说明（撒点 vs 语义）

- **撒点门**（`sprinkleGates` 标的）：`node.gate` 只带 `{sense,mode}`、**不带 reason** → 选点时落 `LOCKED_FALLBACK[sense]` 中性兜底（SPEC §2.3）。所以本文 §2/§3 的 `reason` 是**给作者的意图注释**，真要玩家读到具体成因，得走下面两条之一：
  1. **整潜门**（`diveModifier.gate.reason`·POI 级·C.2）——整片区一句成因。
  2. **手植节点**（若某岔口要专属文案，在事件 / 地图数据里给该 node 显式 `gate.reason`；撒点不覆盖已有 gate）。
- **术语门**（`check-terminology`·SPEC §7）会盯「需要灯 / 需要声呐」等固定键——override 文案别撞固定键的漂移。

## 5. 开放决定（C.1–C.5 · 已填推荐默认）

### C.1 洞穴：按模板 vs 逐 zone · 且 blue_caves 单感(lamp) vs 双感(lamp+sonar)
- **推荐默认**：**按 tag 三档模板**（§3·免 26 座 churn）+ **blue_caves 双感**（旗舰要展示 2×2 四格·§2.1）。
- 「改这里」：想逐 zone 手配 → 把 §3 的模板拆成 26 条 per-zone `gates`（churn 大）；想 blue_caves 只灯 → 删 §2.1 的 sonar 行。

### C.2 整潜声呐门（`diveModifier.gate`）on/off — midwater / vent
- **推荐默认**：**按 SPEC §10.4——整潜门只用 `locked`，`hidden` 仅 per-node**。即：**不**给 midwater/vent 叠整潜声呐门（保留纯 per-node 撒点）；若要叠，只允许 `{sonar,locked}`（可见但锁·带 `reason`），**不允许**整潜 `hidden`。
- 影响面：叠整潜门会触发**海图入口门**（`chart.ts::poiDiveBlock`·SPEC §4）——缺声呐挡下潜。故叠 = 多测一批海图入口用例。
- 「改这里」：要叠 → 在该 zone 的 POI（`chart_pois.json`）挂 `diveModifier.gate:{sense:'sonar',mode:'locked',reason:...}`；不叠＝维持本文（推荐）。

### C.3 hiddenRatio「深⇒更藏」曲线
- **推荐默认**：**是**。`shallow ≈ 0`（浅档全 locked＝诚实预告·让新玩家先看清规则）→ 随深度上升（深档 hidden 比例最高·奖励带灯 / 带声呐深潜的玩家）。
- 「改这里」：想全程 locked（零 hidden 岔口·最诚实）→ 各档 `hiddenRatio:0`；想浅档也藏 → 抬 shallow 的 ratio。

### C.4 弱 zone（old_lighthouse_reef / wreck_graveyard）开 / 不开
- **推荐默认**：**hold（先不开）**（§2.5·浅档为主·增量薄·留作诚实开阔对照）。
- 「改这里」：要开 → 用 §2.5 附的示例模板把它们提到「参与」写法。

### C.5 数值
- **推荐默认**：**全部 defer**（本文只命名档 + 曲线形状·统一调时一次性定；对齐 memory `defer-number-tuning`）。
- 「改这里」：统一调那次，把「无/稀/中/密」映射成具体 0..1，填进各 `gates`。

## 6. 每座参与 zone 的「受影响 playthrough」核对清单

| 激活 zone | 必重核 playthrough / 路径 |
|---|---|
| **通用（任一 zone 激活都过）** | `playthrough-sensors.ts`（灯/声呐门语义·**最关键**）· `playthrough-mapgen-scenarios.ts`（图快照·撒点确定性）· `playthrough-ascent.ts`（空屏→上浮·SPEC §5.1）· `playthrough-stealth.ts`（无灯潜行路径）· `playthrough-sonar.ts`（声呐揭示/门）· `playthrough-stalker.ts`（猎手贴邻 vs 门交互） |
| `blue_caves` | + `playthrough-bluecaves.ts` · `playthrough-cave.ts`（maze 双向·门只挡未访问·来路恒显） · `playthrough-columns.ts` / `playthrough-bands.ts`（深度柱/带·若蓝洞挂柱） |
| `open_midwater` | + `playthrough-bands.ts` · `playthrough-columns.ts` · `playthrough.ts`（主流程含中层） · **若采 C.2**：`playthrough-chart.ts` + `chart.ts` / `dive-start.ts` 入口门用例（`poiDiveBlock`） |
