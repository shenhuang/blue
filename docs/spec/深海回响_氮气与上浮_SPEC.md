## §1 氮气物理模型

### §1.1 状态
- 单一 stat `run.stats.nitrogen`，代表组织氮张力（绝对值，非百分比饱和度）。
- **去掉 0–100 上限**：`clampStats` 对 nitrogen 只保 `≥0`，不再 `≤100`。
- 单房间模型（不做多组织/Bühlmann M 值——守「诚实的钟」易读性 + 离散潜结构用不上）。

### §1.2 上限 ← 深度（线性 · 不封顶）
- `ceiling(d) = 100 · d / D_REF`，线性、**不封顶**（`d > D_REF` 时 nitrogen 可超 100）。
- 替换旧 `100·(1−e^(−d/CEILING_D0))`（凹曲线深处平掉、低估深潜；且 `bands.ts` 明写「不硬编码地板」→ 固定 100 cap 会把未来更深 band 压平，正是要根除的「深端平台」）。
- **深度是氮气的唯一入口**。删除死常数 `PRESSURE_SCALE`（它本就是这条线性代理的半成品残骸）。

### §1.3 逼近与排氮（指数 · 半衰期与深度无关 · 双向同管）
- `next = ceiling + (current − ceiling) · 2^(−Δτ / HALF_TIME)`。
- `HALF_TIME` 与深度无关（组织属性）。深度只抬 `ceiling`（梯度），**不改逼近速率常数**——深处吸得快是因为 gap 大，不是半衰期变小。
- 排氮走**同一条指数**：升浅 → `ceiling` 落到 `current` 以下 → 自动向下逼近。删掉旧 `ascent.ts` 的 flat 5/回合、2/回合。

### §1.4 不封顶的意义
- 张力不封顶、**危险按档次饱和**（减压病最高=致命封顶）。于是过某深度后例行冲破所有减压阈值，想干净上浮要排的氮越来越多，极深近乎排不完 = 机制层长出「单程下行」。

### §1.5 与氧气的关系（共享起因 · 各算各）
- 氧和氮**共享输入（深度、用力）**，但**各算各的、互不读对方的输出**。
- `o2CostMult`（负伤费力呼吸）只乘氧气、**不碰氮气**（惰性气体灌注受限，多吸气不载氮）——保住现有不变量。
- 氮气写口只在白名单文件（守 check-boundaries 规则六）。
- 定案（对抗式讨论敲定，勿重开）：深度加速吸氮走**梯度(ceiling)** 不走速率常数；「耗氧↑→氮↑」**非刚性**（只有真用力才两者同增，经灌注共享起因）；刚性挂**总**耗氧被否（=深度双重计入 + 负伤无中生有）。

## §2 「用力」建模（最终收口）

### §2.1 移动 = 纯时间制（不建模用力耗氧）
- 移动**不做**「快=更费氧」的用力建模。速度滑杆本质 = **这趟花几个回合**。
- 快 = 回合少 = 省氧；慢 = 回合多 = 费氧。氮气纯按 `turns + depth` 走 `stepNitrogen`。
- 洋流 = 只加**耗氧**、不特别加氮。滑杆是**纯**「耗氧 ↔ 减压病」的取舍，**不碰体力**（体力正交）。

### §2.2 战斗 = 按负重档加速
- `combatNitrogenGain(N, d, costOxygenTurns × 负重氮系数)`。
- 负重氮系数按 `equipment.ts::weightTier`（百分比四档：轻 ≤40% / 中 ≤70% / 重 ≤100% / 过载 >100%）取值：
  - **轻 1.2 · 中 1.5 · 重 2.0**（作者定起始值；过载本就封锁战斗，不取值）。
- 替换旧扁平 `COMBAT_NITROGEN_MULT = 1.5`：中档正好复现旧值；轻装比旧省氮；重甲多背一层减压债（与已有的体力倍率、命中补正同源=「负重=更费力」）。

## §3 移动与上浮 UI / UX

### §3.1 节点移动
- **只有「上行」移动的节点能展开速率栏**；横向 / 下行 = 普通「点一下就走」的定值节点，不展开。
- **手风琴**：同一时刻只展开一个；展开新的自动收起旧的（不许每个都摊开）。

### §3.2 上行速率栏（全部收在展开栏内）
- 元素顺序：**速度滑杆（连续）→ 读数（耗氧 / 回合 / 到达后 N₂）→ 减压病预测 → 确认（上浮）**。
- 滑杆连续；拖动时所有读数与减压病预测实时联动。
- 参考实现：`move_ui_mock_v2.html`（本 session 交付并经作者批准=「就是这样」）。注意 mock 的 collapse bug 源于缺一条 `.node:not(.open) .body{display:none}`——正式实现用正常手风琴单开逻辑即无此问题。

### §3.3 取舍的方向性 + 猎手
- **省氧 ↔ 减压病的两难只在上行成立**：上行 快=省氧但少排氮(险↑)、慢=费氧但排氮(安全)。
- 横向 / 下行物理反向（回合少=在深处待得短=攒氮少）→ 快单方面更好、无两难（故不给展开）。
- **猎手 × 走快**：快=少几个 tick=更易甩开。这构成上行的真权衡：**快（省氧 + 甩猎手）⇄ 慢（免减压病）**。

### §3.4 最终上浮（全自动 · 无选择）
- 到上浮口 → **自动**梭哈剩余氧气求最安全的上浮；上岸氧气补满。
- 不是第二套减压机制，只兜「上行没排完」的那部分；overstay 太狠照样重伤 / 死。
- 减压（排氮）的**玩家选择集中在上行的滑杆移动**里，最终破水只是结算。

### §3.5 「不上浮」= 玩家自由
- 玩家可选择不上浮、去做别的（继续探索等）。这是纯粹的能动性，**不**引入「必须保证有氧可捡」的内容依赖。

### §3.6 氮气显示
- 状态栏氮气**只显示数字 + 单位**（`N₂`；确切单位名待定），不用 0–100 血条（已不封顶）。

## §4 减压病判定（连续 · 深度感知）
- 由 `(当前深度, 氮残量, 可投入的氧气)` 连续推算是否/多重减压病，取代旧 `determineBends` 的离散 normal/rushed/emergency 三档。
- 深度要进判定（旧版只有 emergency 读深度）：同氮残量，从更深处硬上更惨（压降大、过饱和狠）。
- 危险按档次饱和（最高=致命），底层氮张力不封顶（决定要排多久 / 是否近乎排不完）。
- 具体公式与阈值（旧 SAFE/ONE/TWO=40/60/80）= 占位，作者调（§6）。

## §5 删除 / 重写清单
- **删**：`nitrogen.ts::PRESSURE_SCALE`（死常数）；扁平 `COMBAT_NITROGEN_MULT`（并入负重氮系数）；`ascent.ts` 的 flat 排氮（5/2）；旧 `AscentView` / `resolveAscent` / `planAscent` / normal-rushed-emergency 模式与其 UI。
- **改**：`nitrogen.ts` ceiling 线性化不封顶；`stepNitrogen` 双向同管保留；`state.ts::clampStats` 去 nitrogen 上限；`combat.ts` 用负重氮系数；`events.ts`+`dive-move.ts` 移动测速（上行滑杆=turns、横下定值、洋流耗氧）；`ascent.ts` 最终上浮自动化 + 减压病连续判定。
- 保留不变量：`stepNitrogen` 的「逐回合 step == 一次性 step(turns)」可组合性（猎手守恒口径）。
